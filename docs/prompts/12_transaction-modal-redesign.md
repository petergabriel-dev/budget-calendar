- [ ] Redesign `TransactionFormSheet` at `features/transactions/presentation/components/TransactionFormSheet.kt` to match the Transaction Modal spec. Before implementing, read the Transaction Modal node from `specs/DESIGN_SYSTEM.pen` using `mcp__pencil__batch_get`. Apply all of the following changes: (1) Replace bare title `Text` with a header `Row(spaceBetween)` containing `Text("New Transaction", style = typography.section)` and an `IconButton(onCancel)` with `Icons.Outlined.Close`; for edit mode use "Edit Transaction". (2) Remove the 3-tab `TabRow`; replace with `BcSegmentedControl(options = listOf("Expense", "Income"), selectedIndex = if INCOME 1 else 0)`. (3) Remove the amount `OutlinedTextField`; replace with `LargeAmountInput(label = "AMOUNT", amount = amountInput, onAmountChange = { amountInput = it; localErrors.remove("amount") })`. (4) Replace the "Category" `OutlinedTextField` with `BcInputGroup(label = "Name", value = categoryInput, placeholder = "e.g. Groceries")`; keep `categoryInput` state and `localErrors["category"]` validation. (5) Wrap Date and Account fields in a `Row(horizontalArrangement = Arrangement.spacedBy(16.dp))`; each uses `BcInputGroup(readOnly = true, modifier = Modifier.weight(1f))`; clicking each opens the date picker or account dropdown respectively. (6) Add `var statusInput by remember(stateKey) { mutableStateOf(initialData?.status ?: TransactionStatus.PENDING) }`. Add a STATUS section: `Text("STATUS", caption style, 1sp letterSpacing)` followed by two `BcCheckboxRow` instances — "Scheduled (Future)" checked when `statusInput == PENDING`, "Paid / Cleared" checked when `statusInput == CONFIRMED`; tapping either sets `statusInput` accordingly (mutually exclusive). (7) Remove the `descriptionInput` state and its `OutlinedTextField`. (8) Remove destination account state and dropdown. (9) Remove the `Row` containing `TextButton("Cancel")` and `Button("Save")`; the CTA and delete button move to a sticky bottom section (see layout below). (10) In the save handler, pass `status = statusInput`; remove TRANSFER validation block; pass `description = null`. (11) **Full-height modal + sticky CTA layout**: pass `sheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true)` and `dragHandle = {}` to `ModalBottomSheet`; wrap all sheet content in an outer `Column(Modifier.fillMaxHeight())`; put all form fields in an inner `Column(Modifier.weight(1f).verticalScroll(rememberScrollState()).padding(24.dp), verticalArrangement = Arrangement.spacedBy(24.dp))`; place the CTA and delete button outside that scrollable column in a non-scrolling `Column(Modifier.padding(horizontal = 24.dp, vertical = 16.dp), verticalArrangement = Arrangement.spacedBy(8.dp))` at the bottom — `BcButton(Primary, fillMaxWidth)` first, then the delete `TextButton` for edit mode. After implementing, re-read the Transaction Modal node and diff against the implementation for visual fidelity before marking done.
