# 15 — Calendar Screen Redesign + Overdue Auto-Transition Fix

Redesigns the Calendar screen to match the Pencil design spec (removes header/search/segmented control/Safe to Spend card, adds End of Month Projection pill, redesigns month navigation header, adds dynamic transaction section header). Also fixes the bug where `MarkOverdueTransactionsUseCase` is never called — wires it at app init and foreground resume.

Shadow specs: `specs/shadow/Calendar.shadow.md`, `specs/shadow/Transaction.shadow.md`
Design file: `specs/DESIGN_SYSTEM.pen`

---

## Phase A — Bug Fix: Wire Overdue Auto-Transition

- [ ] Wire `MarkOverdueTransactionsUseCase` at app initialization. In `App.kt`, use `koinInject<MarkOverdueTransactionsUseCase>()` and call it inside a `LaunchedEffect(Unit)` block with `withContext(Dispatchers.Default)`. The use case already exists and is registered in `KoinModules.kt` (line 128 in `useCaseModule`). It calls `transactionRepository.markOverdueTransactions(nowMillis)` which fetches all PENDING transactions and updates those where `nowMillis > transaction.date + MILLIS_IN_DAY` to OVERDUE. Verify by searching for `MarkOverdueTransactionsUseCase` in `KoinModules.kt` to confirm the factory registration. After wiring, the call should fire once on cold start. (Use Skill: kmp) (Reference: specs/shadow/Transaction.shadow.md)

- [ ] Add foreground lifecycle observer to call `MarkOverdueTransactionsUseCase` on app resume. In `App.kt`, use `LifecycleEventObserver` with `LocalLifecycleOwner.current` to listen for `Lifecycle.Event.ON_RESUME`. On each resume event (after the first, which is covered by the init `LaunchedEffect`), call `markOverdueTransactionsUseCase(DateUtils.nowMillis())` in a coroutine scope. This catches the case where the user backgrounds the app overnight and transactions cross into overdue. The operation is idempotent — already-OVERDUE transactions are excluded by the `getPendingTransactions` query. (Use Skill: kmp) (Reference: specs/shadow/Transaction.shadow.md)

## Phase B — End of Month Projection (Domain Layer)

- [ ] Add `getPendingAndOverdueForSpendingPoolInRange` query to `transactions.sq`. Add the following named query to `src/commonMain/sqldelight/com/petergabriel/budgetcalendar/core/database/transactions.sq`: `getPendingAndOverdueForSpendingPoolInRange :many — SELECT t.* FROM transactions t JOIN accounts a ON t.account_id = a.id WHERE t.status IN ('pending', 'overdue') AND t.date >= :startMillis AND t.date <= :endMillis AND a.is_in_spending_pool = 1 AND t.is_sandbox = 0 ORDER BY t.date ASC;`. This query returns all pending/overdue transactions on spending pool accounts within a date range, used by the projection calculation. Verify the query compiles by running `./gradlew :composeApp:generateCommonMainBudgetCalendarDatabaseInterface` from `src/mobile/`. (Use Skill: kmp) (Reference: specs/shadow/Calendar.shadow.md)

- [ ] Add `getMonthProjectionTransactions` to repository layer. Add method signature `fun getMonthProjectionTransactions(startMillis: Long, endMillis: Long): Flow<List<Transaction>>` to `ITransactionRepository` interface. Implement it in `TransactionRepositoryImpl` by calling the new `getPendingAndOverdueForSpendingPoolInRange` query and mapping results through the existing `TransactionMapper`. Use `asFlow().mapToList(Dispatchers.Default)` pattern matching other repository methods. Search for existing repository methods using `cocoindex-code` with query `"TransactionRepositoryImpl getByDateRange"` to follow the same pattern. (Use Skill: kmp) (Reference: specs/shadow/Calendar.shadow.md)

- [ ] Create `CalculateMonthProjectionUseCase` at `features/calendar/domain/usecase/CalculateMonthProjectionUseCase.kt`. Constructor takes `ITransactionRepository` and `IAccountRepository`. Method `operator fun invoke(yearMonth: YearMonth): Flow<Long>` — combines two flows: (1) `accountRepository.getSpendingPoolAccounts()` → sum balances to get `poolBalance`, (2) `transactionRepository.getMonthProjectionTransactions(startOfMonthMillis, endOfMonthMillis)` → for each transaction, if type is INCOME add amount, if EXPENSE subtract amount → `signedTransactionSum`. Return `poolBalance + signedTransactionSum`. Use `kotlinx.coroutines.flow.combine` to merge the two flows. Convert `yearMonth.firstDay()` and last day to millis using `atStartOfDayIn(TimeZone.UTC).toEpochMilliseconds()`. Handle edge case: if pool has no accounts, emit 0L. (Use Skill: kmp) (Reference: specs/shadow/Calendar.shadow.md)
