# Bug Fix: Computed Account Balance from Transaction History (Option B1)

**Date:** 2026-04-02
**Bug Report:** `bug-reports/account-management/2026-04-02_1200_stored-balance-permanently-drifts-from-transaction-history.md`
**Feature:** Account Management + Safe to Spend + Transaction Management

**Root Cause:** The `accounts.balance` column is a mutable running total dependent on every code path calling `adjustBalance()` correctly. Any missed call silently corrupts the balance with no way to detect or repair it. The `flatMapLatest` fix (prompt 24) makes flows reactive, but re-querying the DB just surfaces the wrong stored value faster.

**Fix:** Replace `balance` (mutable) with `initial_balance` (immutable, set at creation) and add `signed_amount` to transactions. Balance is computed on every read: `initial_balance + SUM(signed_amount WHERE status='confirmed')`. Pre-existing corrupt rows are automatically repaired by this migration.
